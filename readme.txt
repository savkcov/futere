Приложение прдсталяет собой одностраничную web страницу "Комментарии".
При разработке приложения использовались языки прграммирования PHP, SQL, Javascript. Фреймворки на стороне серверной части не использовались (нативный PHP), 
на клиентской части использовался js фреймворк jquery. В качестве СУБД использовалась БД Mysql.
В коде приложения присутствуют комментарии для более быстрого "понимания" логики приложения.


Функционал прлиложения:
	1. Просмотр списка комментариев: Список загружается при входе/обновлении страницы
	2. Добавление комментария. Алгорит добавления комментария: При заполнении формы добавления комментария и нажатия на кнопку "Отправить" сначала производится 
	проверка заполнения формы на клиентской чати, если хотя бы одно поле не заполнено, выдается ошибка, данные формы при этом не отпраляются на сервер.
	Если все заполнено, данные с помощью технологии "Ajax" передаются на сервер, где проходят аналогичную проверку. Далее данные записываются в БД, 
	если по какой-либо причине данные не попали в БД выдается ошибка и пользователь видит ее в диалоговом окне (alert). Если все прошло удачно и данные записались, 
	пользователь видит диалогове окно (alert) с сообщением об удачном добавлении комментария. Далее происходит перезагрузка страницы 
	и комментарий появляется вверху списка.

Описание структуры фаловой системы приложения:
	1. файл index.php - единая точнка входа в приложения
	2. Файл config.php - конфигурационный файл приложения. В нем указаны основные настройки (имя сайта, параметры подключения к БД, пути к директориям).
	3. Файл future.sql - Дамп БД.
	4. Директория classes - Содержит файлы абстрактных классов приложения от которых наследуются классы, реализующие функционал. Также содержится класс
	подключения к БД
	5. Директория controllers - Содержит файлы классов отвечающих за обработку запросов клиента
	6. Директория models - Содержит файлы классов отвечающих за обработку и сохранения данных
	7. Директория src - Содержит js, css и файлы изображений. Файлы необходимы для стилизации и реализации логики приложения на стороне клиента
	8. Директория tpl - Содержит файлы html шаблонов приложения
	9. Директория views - Содержит файлы классов отвечающих за предоставление данных
	
Способ развертывания приложения:
	1. Скопировать файлы и директрии из архива в корневую директорию вашего сайта
	2. Импортировть файл future.sql в БД Mysql вашего сайта
	3. Изменить настройки подключения к БД в файле config.php, для этого поменять значения  следующих контант на актуальные для вашей БД: 
		"db_host" - хост базы данных
		"db_port" -порт для базы данных
		"db_user" - имя пользователя базы данных
		"db_password" - пароль к базе данных
		"db_database" - имя базы данных
